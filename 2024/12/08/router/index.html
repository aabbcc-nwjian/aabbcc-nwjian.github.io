什么是router

概念

React的Router是一个为React设计的路由库，它使得在单页应用(SPA)中能够实现多页面的导航，并且保留了浏览器的历史记录功能。

主要特性

  1. 声明式路由：通过定义组件的方式来配置路由。
  2. 嵌套路由：支持在父路由中嵌套定义子路由。
  3. 动态路由：支持在URL中使用参数，从而实现动态路由匹配。
  4. 程序化导航：允许通过代码控制路由跳转。
  5. 浏览器历史记录：与浏览器的历史记录API集成，支持前进和后退操作。

React Router 允许定义不同的 URL 路径，并根据这些路径渲染相应的组件，从而实现页面的切换。它提供了多个核心组件：BrowserRouter、Route、Link、Switch

React Router核心组件

1. BrowserRouter

  • 作用：BrowserRouter是一个高阶组件，用于包裹整个应用，提供路由功能。它使用HTML5的history API来保持UI和URL的同步。
  • 用法：通常放在应用的最外层，包裹整个应用组件树。

2. Route

  • 作用：Route是用于定义路由规则的组件。它根据当前URL渲染对应的组件。
  • 属性：
    • path：指定匹配的URL路径。
    • component：指定匹配路径时要渲染的组件。
    • render：一个返回React元素的函数，可以用于内联渲染和包装组件。
      render通常用于渲染动态组件，比直接使用component更灵活，它允许你在渲染组件之前进行一些额外的逻辑处理。
  • 用法：通常与Switch组件一起使用，确保一次只渲染一个匹配的路由。

3. Switch

  • 作用：Switch组件用于包裹一组Route组件，一次仅渲染匹配的第一个路由。它确保了路由匹配的独占性，避免渲染多个路由组件。
  • 用法：通常与多个Route组件一起使用，确保只有一个Route被渲染。 在v6版本中，Switch组件已被移除，取而代之的是Routes组件。Routes组件的行为类似于Switch，也只会匹配第一个Route并停止匹配其他Route。Routes组件支持相对路由，使得路径的编写更精简、更可预测。

4. Link

  • 作用：Link组件用于创建导航链接，用户点击链接后，浏览器的URL会更新，同时应用渲染相应的组件。它类似于HTML中的标签，但不会导致页面刷新。
  • 属性：
    • to：指定导航的目标路径，可以是一个字符串，也可以是一个包含路径和查询参数的对象。
  • 用法：在需要进行导航的地方使用Link组件。

两种路由模式

React Router中具有两种路由模式，分别为Hash路由和History路由。

Hash路由

特点：

  • URL中带有#符号（例如：http://example.com/#/user/123）。
  • #及其后面的内容不会被发送到服务器，因此改变hash值不会重新加载页面。
  • 兼容性较好，即使在不支持HTML5 History API的旧版浏览器中也能使用。
  • 无需服务器进行特殊配置。

实现原理：

  • 通过监听浏览器的hashchange事件来感知URL中hash部分的变化。
  • 当hash发生变化时，根据新的hash值来匹配对应的路由并展示相应的组件。

History路由

特点：

  • URL没有#符号（例如：http://example.com/user/123）。
  • 使用HTML5的History API（如pushState和replaceState方法）来改变URL，不会触发页面的刷新。
  • 需要服务器端的配置支持，以便在刷新页面或直接访问某个路由时能够正确地响应。

实现原理：

  • 通过监听浏览器的历史记录变化（pushState、replaceState、popstate）来处理路由变化。
  • 当使用History路由时，浏览器地址栏的URL会真实地反映出当前页面的路由状态。

总的来说，这两种路由模式各有优缺点，开发者应根据具体的应用场景和需求来选择合适的路由模式。
