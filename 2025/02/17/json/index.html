JSON 简介

什么是 JSON？

JSON（JavaScript Object Notation）是一种轻量级的数据交换格式，它是来源于JavaScript的对象语法。简洁和清晰的层次结构使得 JSON 成为理想的数据交换语言。
理解 JSON 最关键的一点是要把它当成一种数据格式，而不是编程语言。

JSON 的特点

易于阅读和编写：JSON 格式的数据结构清晰，易于人类阅读和编写。 易于机器解析和生成：JSON 格式的数据可以被各种编程语言轻松解析和生成。 数据格式简单：与 XML 相比，JSON 遵循简单、紧凑的风格。

JSON 的使用场景

1. Web 开发

  • 前后端数据交互：JSON 常用于前后端之间的数据传输。前端通过 AJAX 请求从后端获取 JSON 数据，并将其渲染到页面上。
  • API 响应：大多数 RESTful API 使用 JSON 作为数据格式，返回结构化的数据供客户端使用。

2. 配置文件

  • 应用程序配置：许多应用程序使用 JSON 文件来存储配置信息，如数据库连接信息、应用设置等。
  • 开发工具配置：如 ESLint、Prettier 等工具使用 JSON 文件来定义规则和配置。

3. 数据存储

  • NoSQL 数据库：如 MongoDB 使用 BSON（Binary JSON）格式存储数据，JSON 是其数据模型的基础。
  • 缓存数据：Redis 等缓存系统支持 JSON 格式的数据存储，方便快速读取和写入。

4. 跨平台数据交换

  • 移动应用：JSON 常用于移动应用与服务器之间的数据交换，支持跨平台开发。
  • 物联网（IoT）：IoT 设备通常使用 JSON 格式进行数据传输，因其轻量且易于解析。

JSON 与 JavaScript 的区别

JSON 不属于 JavaScript，它们只是拥有相同的语法而已。JSON 也不是只能在 JavaScript 中使用，它是一种通用数据格式。很多语言都有解析和序列化 JSON 的内置能力。

JSON 的基本结构

JSON 仅支持以下三种类型的值：

简单值：字符串、数值、布尔值和 null 可以在 JSON 中出现，就像在 JavaScript 中一样。特殊值 undefined 不可以。

对象：第一种复杂数据类型，对象表示有序键/值对。每个值可以是简单值，也可以是复杂类型。

数组：第二种复杂数据类型，数组表示可以通过数值

这四种简单值中不允许内嵌其他值，而复杂数据类型允许内嵌其他值

JSON 支持嵌套结构，即对象中可以包含数组，数组中也可以包含对象。

JSON.stringify()方法

JSON.stringify() 方法用于将 JavaScript 对象或值转换为 JSON 字符串。

语法

参数

  • value: 必需，要转换为 JSON 字符串的值。
  • replacer: 可选，用于转换结果的函数或数组。
    • 如果是一个函数，则每个属性都会经过该函数的转换。
    • 如果是一个数组，则只有包含在数组中的属性名才会被序列化。
  • space: 可选，用于控制输出字符串的缩进。

总结

JSON 是一种轻量级数据格式，可以方便地表示复杂数据结构。这个格式使用 JavaScript 语法的一个子集表示对象、数组、字符串、数值、布尔值和 null。虽然 XML 也能胜任同样的角色，但 JSON 更简洁，JavaScript 支持也更好。更重要的是，所有浏览器都已经原生支持全局 JSON 对象。
